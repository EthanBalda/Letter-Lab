The Letters Rules:

a; Apathetic
Does nothing. Other letters can interact with it.

b; Basic : aba -> aab OR baa
Swaps places with a cardinally adjacent letter.

c; Complex : aacaa -> caaaa OR aaaac
Swaps places with a letter cardinally adjacent letter one slot away from it. One more space than b.

d; Delete : adb -> ad_
Deletes whatever letter tries to interact with it. Has some weird interactions.

e; Eat : abcec -> ab_cc
Eats any adjacent letter, deleting that letter and becoming it.

f; Force : abfcc -> bfcca OR cabfc
Moves the entire string or grid cardinally one space. If a letter would be knocked off the string, it wraps back to the start.

g; Grab : abcagbb -> bcaagbb OR abacgbb OR acabgbb
Grabs a cardinally adjacent letter, pulls it adjacent to ‘g’, then moves all other letters back to fill space.

h; Hold (This may need to be removed with grids) : ahb -> ahb_
Holds a selected letter over its head, effectively removing it from the string or grid. Can swap out whatever is held with another letter. If it grabs a ‘d’, the ‘h’ and whatever it was holding get deleted, and the ‘d’ moves to the ‘h’ ‘s location. Can only hold and swap any adjacent letter.

i; Immovable : acbi -> acbi
Cannot be interacted with by anything other than ‘n’.

j; Jump : abcjcba -> jbcacba OR abcacbj
Like ‘c’, but with 1 more range.

k; Kick : bkabba -> bkbbaa
The exact opposite of ‘g’. Kicks any cardinally adjacent letter to the edge of the grid or string, then shifts every letter towards it to fill in the gap.

l; Lovely (These should probably be ditched if we commit to grids.) : bacblccbb -> bbaclcbbc
Pulls every cardinal letter into it whenever it is selected. Letters that would move to the selected ‘l’s slot instead go to the edges.

m; Monstrous (These should probably be ditched if we commit to grids.) : bacbmccbb -> acbbmbccb
Opposite of ‘l’, they scare every cardinal letter one slot away. Any letters that would go off the grid, instead end up in the empty adjacent slots.
